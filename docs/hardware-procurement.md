# 硬件采购清单 / Hardware Procurement List

> 版本：v1.0  
> 日期：2026-06-29  
> 依据：[technical-plan.md](technical-plan.md) §4、[detailed-design/hardware.md](detailed-design/hardware.md)、
> [mvp-preparation-and-code-scope.md](mvp-preparation-and-code-scope.md)

整合自各设计文档的一份**可直接下单**的采购清单。原则：先按「开发验证」采购开发板/评估板，
跑通最小闭环后再进工程样机，**不要一开始就做量产硬件**。

> 重要决定：**不使用蜂鸣器，本地提示只用 LED**（多模式：熄灭/常亮/循环闪烁/摩斯码，
> 默认 SOS，服务端可调）。详见固件 `firmware/include/lm_led.h`。采购据此**不含蜂鸣器**。

## 推荐主方案（国内优先版 Core v1）

```text
4G Cat 1 bis 通信模组 + 双频多星座 GNSS + 低功耗 IMU + 低功耗 MCU + 锂电/充电/电量计 + LED
```

## 暂缓采购（MVP 不建议先买）

| 硬件 | 暂缓原因 |
| --- | --- |
| 蜂鸣器 | 产品决定本地提示只用 LED |
| RTK 厘米级定位模组 | 成本/功耗/天线/订阅服务都不适合 MVP |
| 极致小型化猫用 Mini 模具 | 先验证定位与续航，再做极限结构 |
| 大规模量产工装 | P0/P1 硬件方案仍会变化 |
| 完整 IP67 量产结构 | 先做简化防水验证，PCB/天线稳定后再投入 |
| eSIM 批量烧录工具 | 小样机用 nano SIM 或少量 eSIM 即可 |

## 接线参考（P0 开发板互联）

```text
电池 -> 充电/保护 -> 电量计 -> 电源轨 -> MCU / Cat 1 bis / GNSS / IMU / LED

MCU <-> Cat 1 bis 模组 : UART（AT 指令 + 数据上报）
MCU <-> GNSS 模组       : UART（NMEA / 厂商二进制）
MCU <-> IMU             : I2C（运动中断唤醒 MCU）
MCU <-> 电量计           : I2C（电量/电压）
MCU  -> LED             : GPIO（状态/寻宠指示，多模式见 lm_led）
```

## 验收口径（采购到位后）

- 开阔户外连续 30 分钟轨迹无明显断点。
- 城市街区轨迹可读，无大面积漂移。
- 日常模式续航 ≥ 3 天；寻宠模式连续 ≥ 6 小时。
- 网络中断后能离线缓存并恢复后补传。
- LED 多模式（含摩斯 SOS）可由固件驱动并经云端配置。
