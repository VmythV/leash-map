# 硬件零部件详解 / Hardware Components Spec

> 版本：v1.0  
> 日期：2026-06-30  
> 依据：[technical-plan.md](technical-plan.md) §4、[detailed-design/hardware.md](detailed-design/hardware.md)、
> [hardware-procurement.md](hardware-procurement.md)

针对**现有 MVP 硬件方案（国内优先版 Core v1）**，逐个零部件说明：**推荐型号 / 作用 / 选型理由 /
接口与对应固件抽象 / 后续拓展点**。

> 产品决定：本地提示**只用 LED，不用蜂鸣器**（多模式：熄灭/常亮/循环闪烁/摩斯码，默认 SOS，
> 服务端可调，见 `firmware/include/lm_led.h`）。

## 系统框图

```text
电池 ── 充电/保护 ── 电量计 ── 电源轨 ┬─ MCU（主控/状态机）
                                      ├─ Cat 1 bis 模组 ── LTE 天线
                                      ├─ 双频 GNSS 模组 ── GNSS 天线
                                      ├─ IMU（运动传感）
                                      └─ LED（状态/寻宠指示）
MCU 接口：UART↔模组、UART↔GNSS、I2C↔IMU、I2C↔电量计、GPIO→LED
```

固件以 `firmware/include/lm_drivers.h` 的 HAL 抽象隔离硬件：
`gnss_read / modem_connect / modem_publish / imu_read_motion / battery_read / clock_now / log_event`。
更换具体型号不影响业务状态机。

## 1. 蜂窝通信模组（4G Cat 1 bis）

- **推荐型号**：Quectel **EG915Q** 系列（国内主力）；可选 **EG800K / EG810M** 同级，按供应链与认证选型。
- **作用**：设备联网主链路，向云端上报心跳/定位/事件（MQTT over TLS 优先，HTTPS 兜底）。
- **选型理由**：Cat 1 bis 在功耗、成本、移动性、覆盖、实时性之间最平衡，适合宠物这种持续移动场景；比 NB-IoT 移动性/时延好，比 Cat 4 便宜省电。
- **接口/固件**：UART（AT 指令 + 数据）→ `modem_connect()` / `modem_publish()`。
- **后续拓展点**：
  - 海外版换 **LTE-M/NB-IoT/NTN SiP（Nordic nRF9151）**，集成度高、超低功耗资产追踪。
  - 选**自带 GNSS 的 Cat 1 模组**，实测达标后可省掉独立 GNSS 模组、降本降体积。
  - 远期评估 **5G RedCap**；卫星 NTN 兜底无蜂窝覆盖区域。

## 2. GNSS 定位模组（双频多星座）

- **推荐型号**：Quectel **LC29H**（双频 L1+L5，多星座 GPS/BDS/Galileo/GLONASS/QZSS/NavIC）；对比基准可加 **LC76G**。
- **作用**：主定位来源，输出经纬度/速度/航向/精度，支撑轨迹与围栏。
- **选型理由**：双频比单频在城市楼宇反射场景显著降低多路径误差；多星座提升可用性与首次定位速度。
- **接口/固件**：UART（NMEA 或厂商二进制）→ `gnss_read()`（返回 fix）。
- **后续拓展点**：
  - **AGNSS / 星历预下载**（云端下发）加速冷启动 TTFF。
  - **航位推算（DR/组合惯导）**：隧道/地库无星时用 IMU 续推。
  - LC29H 有 **RTK 变体**，远期高精度需求可升级（但需基站/订阅，MVP 不上）。
  - GNSS 失败时多源降级：基站 / Wi-Fi Scan / BLE 近距（见第 11 节）。

## 3. 主控 MCU

- **推荐型号**：**STM32U0 / STM32L4**（超低功耗 Cortex-M），或 **Nordic nRF52**（带 BLE）。
- **作用**：运行固件状态机（9 态）、外设采集、低功耗调度、离线缓存、上报协议封装、OTA。
- **选型理由**：MVP 选团队最熟悉的低功耗 MCU，不过早追极限小型化；nRF52 额外带 BLE，利于配网与近距省电。
- **接口/固件**：承载整个 `lm_app` 主循环与 `lm_state`/`lm_cache`/`lm_protocol`/`lm_ota`。
- **后续拓展点**：
  - 选 **nRF52/nRF53** 解锁 BLE 配网、靠近省电、近距寻宠。
  - 更大 Flash 支持 **双 bank OTA**（A/B 回滚，`lm_ota` 已建模）。
  - 带 **安全元件/TrustZone（STM32L5/U5）** 存设备证书与密钥。

## 4. 运动传感器（低功耗 IMU）

- **推荐型号**：ST **LIS2DW12**（超低功耗三轴加速度计）。
- **作用**：运动唤醒、静止判断、活动量统计；驱动功耗模式切换（静止→省电，运动→追踪）。
- **选型理由**：纳安级待机、运动中断可唤醒 MCU，避免持续高功耗定位；是续航策略的关键。
- **接口/固件**：I2C（运动中断）→ `imu_read_motion()`（still/moving）。
- **后续拓展点**：
  - 升级 **6 轴 IMU（含陀螺，如 LSM6DSOX）**：姿态/行为分析、佩戴方向校正。
  - 利用 **机器学习核（MLC）**：在传感器端做活动分类（走/跑/玩/休息），喂给云端运动分析。
  - 摔倒/异常静止检测，扩展健康类告警。

## 5. 电量计（Fuel Gauge）

- **推荐型号**：**MAX17048**（轻量 ModelGauge）或 **BQ27xxx**（库仑计，更精确）。
- **作用**：电量百分比与电压估算，支撑低电告警（30/15/5% 阈值）与续航测试。
- **选型理由**：低电提醒与续航结论都需要可靠电量数据，纯分压估算误差大。
- **接口/固件**：I2C → `battery_read()`（0–100%）。
- **后续拓展点**：
  - 换 **BQ27xxx 库仑计**做更准的容量/老化估计与温度补偿。
  - 上报充电状态/温度，App 区分「充电中」。

## 6. 充电与电池保护

- **推荐型号**：单节锂电**充电管理 IC（如 TP4056/BQ2403x 同级）+ 保护板**（过充/过放/过流/温度保护）。
- **作用**：安全充电与电池保护，样机阶段用模块、量产再集成。
- **选型理由**：宠物设备佩戴环境复杂，温度与过放保护是可靠性底线。
- **后续拓展点**：
  - **磁吸触点 / 隐藏式充电**，减少外露接口、利于防水（防水与充电冲突是已知风险）。
  - 评估 **Qi 无线充电**底座，进一步免接口。

## 7. 电池

- **推荐型号**：**500–700mAh 单节锂聚合物（LiPo）**，容量接近目标成品。
- **作用**：供电与续航；容量必须接近成品，否则功耗结论不可靠。
- **选型理由**：在重量（Core ≤35g）与续航（日常 3–7 天）之间取平衡。
- **后续拓展点**：
  - **高能量密度电芯**提升续航或缩体积。
  - 猫用 **Mini 版**用更小电芯（重量/天线成为第一约束）。

## 8. 天线（LTE / GNSS）

- **推荐型号**：LTE **FPC/PCB 天线**、GNSS **有源/无源天线**（陶瓷贴片），配 IPEX/U.FL 转接线。
- **作用**：通信与定位的射频前端；天线位置直接决定联网与定位质量。
- **选型理由**：小体积天线效果不足是头号风险，必须在 EVT 阶段重点测佩戴姿态与外壳前后对比。
- **后续拓展点**：
  - 针对外壳定制 **专用天线结构**；GNSS 有源天线 + LNA 提升弱信号。
  - 远期 LTE/GNSS/BLE **多合一/共形天线**减体积。

## 9. 指示灯 LED（无蜂鸣器）

- **推荐型号**：单色/双色贴片或直插 **LED + 限流电阻**（可选导光柱用于防水透光）。
- **作用**：设备状态与寻宠提示。固件支持 4 种模式：熄灭/常亮/循环闪烁/**摩斯码**（默认 SOS，云端可改消息）。
- **选型理由**：产品决定不使用蜂鸣器（减噪、降功耗、利于防水），用 LED 多模式表达状态。
- **接口/固件**：GPIO → `lm_led_on(elapsed)`（纯逻辑，摩斯编码见 `firmware/src/lm_led.c`）。
- **后续拓展点**：
  - **RGB LED** 用颜色编码状态（绿在线/黄低电/红离区/蓝寻宠）。
  - 高亮 LED + 导光柱，户外/夜间可见性；与寻宠模式联动加强。

## 10. SIM / eSIM

- **推荐型号**：样机用 **nano SIM**（降低调试成本）；量产建议 **MFF2 eSIM**。
- **作用**：蜂窝身份与流量；建议至少两家运营商测试卡避免单网误判。
- **后续拓展点**：
  - **eSIM 批量烧录 + 远程开卡/换卡**；多 IMSI 智能选网。
  - 远期 **iSIM（集成进模组/SoC）** 进一步省体积。

## 11. （可选）近距通信 BLE

- **推荐型号**：随 **nRF52** MCU 内置（无需独立芯片）。
- **作用**：配网、主人手机靠近时省电、近距离寻宠辅助。
- **选型理由**：MVP 可不依赖，但选 nRF52 即「免费」获得；为后续众包找回/近距精确寻宠铺路。
- **后续拓展点**：
  - 类 AirTag 的 **BLE 众包找回网络**作为蜂窝兜底。
  - **UWB** 近距厘米级寻宠（远期高端形态）。

## 组件 → 固件抽象 → 云端能力 对照

| 硬件 | 固件 HAL | 启用的云端/App 能力 |
| --- | --- | --- |
| Cat 1 bis 模组 | `modem_connect` / `modem_publish` | 上行接入、实时推送、下行命令 |
| GNSS 模组 | `gnss_read` | 实时位置、轨迹、围栏、运动分析 |
| IMU | `imu_read_motion` | 功耗模式切换、活动时长/常去区域 |
| 电量计 | `battery_read` | 电量显示、低电告警（每设备） |
| LED | `lm_led_on` | 设备状态/寻宠提示（云端配 LED 模式/摩斯） |
| MCU | 承载 `lm_app/lm_state/lm_ota` | 状态机、离线补传、OTA、配置下发 |

## 拓展路线小结

| 维度 | MVP（现在） | 下一步 | 远期 |
| --- | --- | --- | --- |
| 通信 | Cat 1 bis | 海外 LTE-M/NB-IoT（nRF9151） | 5G RedCap / 卫星 NTN |
| 定位 | 双频 GNSS | AGNSS + 惯导组合 | RTK 高精度 |
| 主控 | STM32/nRF52 | nRF52 + BLE 配网/省电 | 安全元件 + UWB |
| 传感 | 三轴加速度 | 6 轴 + MLC 活动分类 | 行为/健康分析 |
| 提示 | 单/双色 LED | RGB 状态编码 | 高亮导光 + 寻宠联动 |
| 充电 | 模块充电 | 磁吸触点 | 无线充电 |
| 找回 | 蜂窝 GNSS | BLE 近距辅助 | 众包网络 / UWB |

> 型号、价格、供货以供应商/代理商报价为准；切换型号前确认认证、供货周期与最小起订量。
